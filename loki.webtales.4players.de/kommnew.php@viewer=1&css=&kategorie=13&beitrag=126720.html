Iga Pelogran (RIP) So ist es momentan:
Hat man eine Stadt- oder Charbeschreibunt auf HTML-Code aufgebaut und das Häkchen dort gesetzt wird dem Quellcode ein SW HTML TAG vorangestellt beim Speichern.
Ändert man die Beschreibungmuss man den Tag entfernen und den HTML Haken wieder setzen.
So soll es werden:
Entweder:
1. Der TAG wird nicht mehr sichtbar vor die Beschreibung gesetzt oder
2. Ist der TAG in der Stadtbeschreibung wird der Haken bei HTML nicht benötigt.

Vorteile für das Spiel:
Man vergisst nicht ausversehen das Haken setzen und SWHTMLTAG rauszunehmen. Ist der TAG nicht weg, wird er als Text innerhalb der Beschribung gezeigt. ist der Haken nicht gesetzt, wenrden die Tags nicht verarbeitet.
Nachteile für das Spiel:
Keine, da nur optische Verbesserung bzw. Eingabehilfe
Mögliche Problematiken:
Keine!
