Jolina (RIP) So ist es momentan:
Hab seit Jahren schon ein Grauen, wenn ich mir den Quelltext von Scherbenwelten spaßeshalber ansehe.
Hab mal nen Beispiel aus der city.php gebastelt, stark vereinfacht und als reinen Text abgespeichert(UTF-8):

http://hosting.mizukage.de/verbrauch_derzeit.txt
So soll es werden:
Hier meine relativ einfache Überarbeitung, in dem Fall sollte es sogar einfach umzusetzen sein:

http://hosting.mizukage.de/verbrauch_beispiel.txt

mein größter Wunsch:
- obsoleten Tags entfernen
- mehr CSS verwenden
- mehr id-Attribute für einzigartige Bereiche

nice to have

- kleine Hilfen wie Mausover-Effekte, um einzelne Zeilen in einer den schönen, aber teilweise unübersichtlichen Tabellen hervorzuheben
- auch schön wäre noch ein externer Verweis auf eine lokal beim Nutzer gespeicherten css-Datei, wenn er das Grafikpaket nutzt
Vorteile für das Spiel:
* Browser(dat wat Internet anzeigt ;)) wissen gleich, womit sie es zu tun haben,
* weniger Daten zu übertragen,
* individuellere Anpassbarkeit: einfachere Programmierung von S.cripten (Stichwort Greasemonkey), um sich die Website so anzupassen, wie man sie gerne selber haben möchte (Ich rede hier nicht über Spielereien wie Klickbots ...)
und
* bessere Usability/Nutzerfreundlichkeit (Kleinvieh macht auch Mist)
Nachteile für das Spiel:
kein Erkennbarer für mich (kenn ja die Quelle nicht), ausser Umsetzungs-/Tippfehler und natürlich den Aufwand (zBsp: die Werte im css rauszufinden die im Font-Tag etwas anders sind)
Mögliche Problematiken:
Dass das Team an einer XHTML5 Version bereits arbeitet und keine Zeit in den XHTML-HTML3.2-HTML4.01-Mischmasch stecken will.
Sonstiges:
Wenn man die beiden Dateien vergleicht, sieht man nicht sofort von der Größe einen Unterschied. Durch die Anmerkungen ist die Überarbeitete Version sogar größer.
Durch das Weglassen der alten obsoleten Tags und der klugen zentralen Formatvorgabe schrumpt eine große Datei teilweise auf unter 60% gegenüber der derzeitigen HTML-Formatierung von SW.
