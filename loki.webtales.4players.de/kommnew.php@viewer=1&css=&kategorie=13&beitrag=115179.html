Argolas (RIP) So ist es momentan:
Jeder kann Account-Hacking betreiben - Je nachdem wie er Lust/Laune und Zeit hat! Das Passwort kann beliebig oft "falsch" eingegeben werden.
So soll es werden:
Nach der 3. Fehleingabe des Passwortes wird an die bei der Registrierung - unter Stellung/Optionen - genannte E-Mailadresse eine vom System generiertes Passwort per Mail zugesandt.

Dieses neue Passwort muss dann innerhalb von einer gesetzten Frist von 7 - 14 Tagen geändert werden (bei Urlaubsvertretungen, kann die Frist länger eingestellt werden, bzw. der Accountbesitzer wird so oder so informiert! Der kümmert sich sicher schon darum um zeitnah ein neues Passwort einzusetzen)

Text "aus IP ....... wurde 3x das Passwort falsch angegeben - Verdacht auf Hackversuch"



Vorteile für das Spiel:
Hier kann der Account-Besitzer sich dann wie gewohnt einloggen und noch wichtiger - Dieser weiss dass jemand an sein Account will/wollte und wird sich sicherlich ein noch sicheres Passwort, als er schon hat ausdenken.

Account-Hacking scheint ja langsam zum Volkssport zu mutieren und so könnte dem etwas entgegengekommen werden.
Nachteile für das Spiel:
Das Account-Hacking wird schwieriger

Dieser neuer Volkssport wird weniger gefördert

Miesmuschel (RIP) Zipfelmütz, wenn wirklich jemand ein Bot-Netz mit 100k Bots auf meinen Account loslassen will, kann ich ihm auch nicht mehr helfen...

Die andere Frage ist die, ob ein Hacker es da auf einen bestimmten Account abgesehen hat, oder ob er nur irgendeinen Account haben will. Im zweiten Fall probiert er nämlich nicht an einem Account mehrere Passwörter aus, sondern nimmt ein gängiges Passwort und probiert damit alle Accounts durch. Damit umgeht er dann alle Beschränkungen, die den Zugriff auf einen einzelnen Account erschweren sollen. Das sollte man im Hinterkopf behalten, wenn man Verfahren zur Verhinderung von Hacking aufbaut.

Aber unabhängig davon vermute ich mal stark, dass die meisten Fälle von Account-Klau in SW nicht per Brute-Force von statten gehen, sondern durch die Weitergabe von Passwörtern für das Sitten. Von daher halte ich meinen Vorschlag eines eigenen Sitting-Passworts nach wie vor für nützlicher als zeitliche Einschränkungen bei Fehlversuchen. 		12.03.09 20:46
