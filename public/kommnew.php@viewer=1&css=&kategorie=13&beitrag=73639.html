Nalon din Adun (RIP) Ist vermutlich technisch bedingt.

PP in Armee ja oder nein ist eine bool-Variable, relativ klein, nimmt wenig Platz weg etc.

Ändert man das auf 15 PP maximal pro Tag, muss man eine Integer mitschleppen, die größer ist und umfangreichere Abfragemethoden erfordert als die schlichte bool-Variable.

Klingt im ersten Moment wenig, aber wenn man das auf geschätzt 5000 verschiedene Armeen (eher mehr, wir sind ja bald oder schon bei der 11.000. Armee ;) ) hochrechnet, wird es verdammt viel Speicherplatz mehr für die Armeen (Vergleich: bool 1 bit, integer 8 bit (eventuell 4, wenn short verfügbar)) plus die zusätzliche Abfrage auf die bereits eingesetzten PP.

Könnte mittlerweile egal sein, da für SW ja bezahlt wird und daher entsprechend starke Maschinen verfügbar sind, aber früher war das sicher mit ein Grund neben, so ists einfach einfacher *g* 		12.12.05 13:12
Gueldensteining (RIP) Nur mal als technische Anmerkung:
Bei MySQL werden BOOL Werte technisch wie TINYINT Werte behandelt.
Das bedeutet das man immer 8 Bit braucht.
BOOL funktioniert hier so das 0 für FALSE steht und nicht 0 Werte für TRUE stehen.
Heisst im Klartext das man genauso gut ein UNSIGNED TINYINT nehmen kann und damit von 0 bis 255 Platz hat ohne das Speichertechnische Unterschiede da sind.
(Oder man nimmt ein SIGNED, also normales TINYINT und hat -127 bis 127 Platz) 		12.12.05 14:11
