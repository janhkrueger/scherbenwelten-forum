Schadensfaktor = Min (2* (Max-EW/aktuelle EW);1)

Emil Vörshagen (RIP) So ich habe mir jetzt nochmal meine Gedanken zu dem Problem gemacht und folgendes ist dabei herausgekommen:

1) Jedern Tag nach der der Milizsteigerung werden folgende Berechnungen durchgeführt und die Zahlen gespeichert:
- Abhängig von Gebäudeanzahl, Anzahl Milizen und Einwohner wird die Anzahl der Angriffe berechnet, die es brauchen würde, bis keine Milizen mehr da sind.
- Es wird der Durchschnittsmilizverlust pro Angriff berechnet: Anzahl Milizen / Anzahl Angriffe
- Es wird der Durchschnittseinwohnerverlust pro Angriff berechnet (unter Berücksichtigung der 20% Maximalverlust): (Anzahl Einwohner * 0,2) / Anzahl Angriffe



Kommt des dann zu einem Angriff finden pro Angriff folgende Berechnungen statt:

- Miliztode = Anzahl Miliz / Anzahl Gebäude
- Unterschied zu Durchschnittsmilizverlust in Prozent = Miliztode / Durchschnittsmilizverlust
- Einwohnerverlust = Durschschnittseinwohnerverlust * Unterschied zu Durchschnittsmilizverlust in Prozent
- Anzahl Miliz = Anzahl Miliz - Miliztode
- Anzahl Einwohner = Anzahl Einwohner - Einwohnerverlust

Damit kommt man ziemlich genau auf die 20% Einwohnerverlust wenn keine Milizen mehr da sind.

Es gibt bei der ganzen Sache allerdings ein Problem. Wenn der Angriff zum Zeitpunkt der Milizauswertung erfolgt und vorher zB 50 Angriffe erfolgen und danch weitere 50, so steigt der Einwohnerverlust über die 20%, weil die Durchschnittswerte anhand der aktuellen Zahlen neu berechnet werden.

}

echo "Angriffe: $attackCount<br/>";

while ($miliz > 0)
{
$i++;

$percent = 0;

}
