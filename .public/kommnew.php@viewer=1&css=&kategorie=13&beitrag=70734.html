Edward (RIP) So ist es momentan:
Man verfasst einen kleinen Text, vergibt einen Titel und gut ist.
So soll es werden:
Neue Felder zum Klassifizieren des Supportbeitrags.

Priority (Listbox): Low, Medium, High

Type (Listbox): Bug Report, Question, Quest, Help needed, Change Request, Feature Request

Classification (Listbox): Minor, Major

Status (Der Benutzer sieht nur open): Open, Done

Der Benutzer soll die Möglichkeit haben, diese Werte einmalig pro Thema zu setzen um sein Anliegen zu klassifizieren. Der Support kann die Werte ändern. Diese Änderungen werden dem Fragenden dann angezeigt.
Vorteile für das Spiel:
1. Der Spieler kann zum einen einstellen für wie wichtig er seine Frage hält.

2. Der Support kann angeben wie wichtig es wirklich ist. Somit weiß man, wenn es mal länger dauern könnte.

3. Mir zumindest geht es so, dass ich ab und an einen Bug im Progrmm entdecke, den ich melde, aber eigentlich für unwichtig halte. Trotzdem beschäftigt sich der Support sehr schnell damit, als sei es was sehr wichtiges. Um den Jungs die Arbeit zu erleichtern fände ich dieses System sinnvoll. dann weiß der Support sofort, dass man zwar was gefunden hat, was irgendwann mal, wenn etwas Ruhe ist gefixed werden soll, der Spieler selbst aber weiß, dass es nicht eilt und auch nicht auf eien schnelle Reaktion hofft.

4. Dadurch, dass der Support die Tickets selber nachträglich Kategoriesiren kann, können User ablesen, dass ihr Problem zur Kenntnis genommen wurde und als Wichtig oder unwichtig eingestuft wurde. Das Texte schreiben wie "das könnte etwas länger Dauern" entfällt.

5. Es wird für beide Seiten transparenter, was gerade im Supportbereich passiert.
Nachteile für das Spiel:
1. Einige werden sich auf den Schlips getreten fühlen,w eil ihre Probleme runter gestuft werden.

2. Einige Benutzer werden sich überfordert fühlen.

3. Manche werden immer alles auf dringend und besonders wichtig setzen (die ginbts immer und überall)
Mögliche Problematiken:
Neue Programmierung

Größere Datenbank

Länfere Zugriffszeiten auf die DB

Höherer Verwaltungsaufwand für die Software

Benutzer und Support könnten das frisch eingeführte System boykottieren.
